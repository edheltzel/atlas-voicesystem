## Risks

1. **Symlink edit trap** ‚Äî `~/.claude/PAI/USER/Voice` and `claudecode/.claude/PAI/USER/Voice` are the *same files*. Don't expect divergent copies.
2. **MIGRATIONS.md is live-modified + `scripts/restore-hooks.ts` is untracked.** The working tree is mid-change on the fragile PAI-re-apply surface. Diff `git diff MIGRATIONS.md` and read `scripts/restore-hooks.ts` before committing or you may bury intent.
3. **Stow into `~/.claude/PAI` conflicts with `atlas-config`** (memory loa#44). Don't re-run a naive `stow`; the directory-symlink arrangement is deliberate.
4. **`master` push is blocked locally only** ‚Äî `.githooks/pre-push`, installed via `git config core.hooksPath .githooks`. Release path is `gh pr create --base master --head dev`. Never push master directly.
5. **No PAI imports in `server.ts`** (convention) ‚Äî the server is being universalized (issue #1). New host-specific logic belongs in hooks/adapters, not the daemon.
6. **Plans live in the wrong place.** Root `Plans/` and `Voice/Plans/` exist but are **empty**, and both violate the mandatory `.agents/atlas/plans/` rule. Any plan you write goes in `.agents/atlas/plans/`, not these.
7. **Bun only.** `npm`/`npx`/`node` cause silent breakage per conventions.
8. **`/notify` contract is load-bearing** ‚Äî many callers depend on the body shape and 200/400/500 semantics; don't change it without a compat version bump.
